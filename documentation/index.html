Getting Started

Fixton - Crafting Confidence in Repair and Maintenance Next.js Template.

Welcome to Fixton, a responsive, mobile-friendly NextJS 14+, ReactJS 18+ and TypeScript 5+ template powered by Tailwind CSS and Shadcn UI. Fixton is designed for smooth SEO optimization, ensuring that your website works well on all devices.

Folder Structure

The Fixton template has various files and folders you can easily customize. You can change the logo, title, styles, colors, and TypeScript, or add new pages as needed.

Let's discuss about folders and files.

Page and Layouts

All the pages inside /src/app folders

Components

All the reuseable components inside /src/components

Hooks

All the custom hooks inside /src/hooks folders

Mock Data

All the mock data inside /src/lib/mock-data.ts file. The /utils.ts file contains the utility functions.

Middleware

All the middleware functions inside /src/middleware.ts file

components.json file

Configuration of shadcn/ui - Beautifully designed components that you can copy and paste into your apps.

tailwind.config.js:

The tailwind.config.js file holds the TailwindCSS settings for the template. You can customize it as needed. It's included in the template but doesn't need to be uploaded to the server unless you want to modify TailwindCSS classes.

tsconfig.ts:

The /tsconfig.ts file holds the TypeScript settings for the template. You can customize it if needed. It's included but doesn't need to be uploaded to the server and can be deleted if you're not using TypeScript.

Installation

To install the Fixton template, follow the steps below. The template use NextJs 14+, so make sure you have installed NodeJs 18+. Please follow the NextJs installation guide.

Install dependencies

# Install dependencies using npm
npm install
# using bunjs
bun Install
# using yarn
yarn install
# using pnpm
pnpm install

Run Development Server

# Run development server
# using npm
npm run dev
# using bunjs
bun run dev
# using yarn
yarn dev
# using pnpm
pnpm run dev

Make Production Build

# Run production build
# using npm
npm run build
# using bunjs
bun --bun run build
# using yarn
yarn build
# using pnpm
pnpm build

Run Production Server

# Run development server
# using npm
npm start
# using bunjs
bun start
# using yarn
yarn start
# using pnpm
pnpm start
Production Server Development server runs on port 3000. The development server only for development. Do not use it on production. For production, you need to build the project and run the production server.
